------------------------------------
5. Cas critiques, impromptus et vices de Python
------------------------------------

Test 11 : Le cas de False. 
Comme pour True, False vaut logiquement 0 en Python (isinstance(False, int) est vrai). Si ton code ne vérifie pas strictement le type booléen, False passera incognito comme un carré de taille 0 au lieu de lever une erreur !
>>> print_square(False)
Traceback (most recent call last):
TypeError: size must be an integer

Test 12 : Le cas du float 0.0. 
Il vaut zéro, il n'est pas inférieur à zéro, mais c'est un float. Il doit lever une TypeError.
>>> print_square(0.0)
Traceback (most recent call last):
TypeError: size must be an integer

Test 13 : L'infini numérique (float('inf')).
C'est techniquement un float, mais sa valeur dépasse l'entendement. Il doit être bloqué par la vérification de type.
>>> print_square(float('inf'))
Traceback (most recent call last):

Test 14 : Les nombres complexes (ex: 3 + 4j).
C'est un type numérique natif de Python (`complex`), mais ce n'est pas un entier.
