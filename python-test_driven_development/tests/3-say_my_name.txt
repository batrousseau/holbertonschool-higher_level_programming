------------------------------------
1. Le piège absolu des espaces invisibles (Doctest Quirks)
------------------------------------

Test 1 : Le nom de famille est omis (valeur par défaut ""). 
La fonction doit afficher "My name is Walter ". Note bien l'espace obligatoire juste après Walter !
>>> say_my_name("Walter")
My name is Walter 

Test 2 : Les deux arguments sont des chaînes vides.
L'affichage génère "My name is  " (avec exactement deux espaces consécutifs après le "is").
>>> say_my_name("", "")
My name is

2. Le piège des faux-amis du type String
------------------------------------

TypeError: first_name must be a string

Test 4 : Le piège du Booléen. En Python, True et False appartiennent à la famille des entiers. Tu dois explicitement vérifier leur type.
>>> say_my_name("Walter", True)

TypeError: last_name must be a string

Test 5 : Passage d'un nombre sous forme d'entier pur.
>>> say_my_name(123, "White")
